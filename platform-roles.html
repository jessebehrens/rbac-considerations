Platform Administration Roles

Design roles for Platform Admins and FinOps who need administrative access without data privileges.

The Platform Team Challenge

Your platform team needs to manage Snowflake—warehouses, resource monitors, integrations, and more—but they shouldn't necessarily have access to all business data. How do you grant administrative capabilities without granting data access?

Platform Admin

Manage infrastructure: warehouses, resource monitors, network policies, integrations.

FinOps Admin

Monitor costs, set budgets, analyze usage patterns. No data access needed.

Platform Admin Role

The Platform Admin manages Snowflake infrastructure but doesn't need to read business data:

-- Create the Platform Admin functional role
USE ROLE USERADMIN;
CREATE ROLE PLATFORM_ADMIN
    COMMENT = 'Platform administrators - infrastructure management';

USE ROLE SECURITYADMIN;

-- Warehouse management
GRANT CREATE WAREHOUSE ON ACCOUNT TO ROLE PLATFORM_ADMIN;
GRANT MANAGE WAREHOUSES ON ACCOUNT TO ROLE PLATFORM_ADMIN;

-- Resource monitors (for cost control)
GRANT CREATE RESOURCE MONITOR ON ACCOUNT TO ROLE PLATFORM_ADMIN;
GRANT MODIFY ON ALL RESOURCE MONITORS IN ACCOUNT TO ROLE PLATFORM_ADMIN;

-- Integrations (storage, API, notification)
GRANT CREATE INTEGRATION ON ACCOUNT TO ROLE PLATFORM_ADMIN;

-- Network policies
GRANT CREATE NETWORK POLICY ON ACCOUNT TO ROLE PLATFORM_ADMIN;

-- Database creation (without automatic data access)
GRANT CREATE DATABASE ON ACCOUNT TO ROLE PLATFORM_ADMIN;

-- Monitor account usage
GRANT IMPORTED PRIVILEGES ON DATABASE SNOWFLAKE TO ROLE PLATFORM_ADMIN;

-- Roll up to SECURITYADMIN (has similar privileges minus MANAGE GRANTS)
GRANT ROLE PLATFORM_ADMIN TO ROLE SECURITYADMIN;
Why SECURITYADMIN?

PLATFORM_ADMIN has privileges similar to SECURITYADMIN but without MANAGE GRANTS. Rolling up to SECURITYADMIN keeps powerful admin roles consolidated and avoids daily use of SECURITYADMIN directly.

FinOps Admin Role

The FinOps Admin monitors costs and usage without accessing data. They need read access to billing and usage views:

-- Create the FinOps Admin functional role
USE ROLE USERADMIN;
CREATE ROLE FINOPS_ADMIN
    COMMENT = 'FinOps team - cost monitoring and optimization';

USE ROLE SECURITYADMIN;

-- Access to SNOWFLAKE database for usage/billing data
GRANT IMPORTED PRIVILEGES ON DATABASE SNOWFLAKE TO ROLE FINOPS_ADMIN;

-- Resource monitors (view and modify credits)
GRANT MODIFY ON ALL RESOURCE MONITORS IN ACCOUNT TO ROLE FINOPS_ADMIN;

-- View warehouses (to understand compute costs)
GRANT MONITOR ON ALL WAREHOUSES IN ACCOUNT TO ROLE FINOPS_ADMIN;

-- A dedicated warehouse for running cost queries
CREATE WAREHOUSE IF NOT EXISTS FINOPS_WH 
    WAREHOUSE_SIZE = XSMALL
    AUTO_SUSPEND = 60
    AUTO_RESUME = TRUE;
GRANT USAGE ON WAREHOUSE FINOPS_WH TO ROLE FINOPS_ADMIN;

-- Roll up to ACCOUNTADMIN (billing/cost management)
GRANT ROLE FINOPS_ADMIN TO ROLE ACCOUNTADMIN;

Common FinOps Queries

-- Credit consumption by warehouse (FinOps can run these)
SELECT 
    WAREHOUSE_NAME,
    SUM(CREDITS_USED) AS TOTAL_CREDITS,
    SUM(CREDITS_USED_COMPUTE) AS COMPUTE_CREDITS,
    SUM(CREDITS_USED_CLOUD_SERVICES) AS CLOUD_SERVICES_CREDITS
FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
WHERE START_TIME >= DATEADD(MONTH, -1, CURRENT_TIMESTAMP())
GROUP BY WAREHOUSE_NAME
ORDER BY TOTAL_CREDITS DESC;

-- Storage costs by database
SELECT 
    DATABASE_NAME,
    AVG(AVERAGE_DATABASE_BYTES) / POWER(1024, 4) AS AVG_TB,
    MAX(AVERAGE_DATABASE_BYTES) / POWER(1024, 4) AS MAX_TB
FROM SNOWFLAKE.ACCOUNT_USAGE.DATABASE_STORAGE_USAGE_HISTORY
WHERE USAGE_DATE >= DATEADD(MONTH, -1, CURRENT_DATE())
GROUP BY DATABASE_NAME
ORDER BY AVG_TB DESC;

Platform Role Hierarchy

Hierarchy Decision

FinOps Admin rolls up to ACCOUNTADMIN (billing and cost management). Platform Admin rolls up to SECURITYADMIN (infrastructure admin without MANAGE GRANTS).
